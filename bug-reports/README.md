# Bug reports

Esta carpeta guarda **plantillas y ejemplos** de reportes de bug. En un proyecto real, esto suele vivir en Jira/Linear, pero para tu portfolio está buenísimo mostrar que sabes **documentar**.

## Cómo usar
- Copia `bug_report_template.md`
- Completa los campos con información concreta
- Adjunta evidencia (logs, request/response, capturas)

## Qué evalúan los recruiters / TLs
- Claridad del **resultado actual** vs **resultado esperado**
- Pasos reproducibles (otra persona puede reproducirlo)
- Severidad y prioridad razonables
- Evidencia técnica (especialmente en API)
